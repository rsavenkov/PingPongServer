Ping'nPong сервер слушает по порту 80, отвечает с кодом 200 только на url /handler. В остальных случаях код ответа 400

Описание струкруры каталога:
  hsqldb                         - дистрибутив hsqldb базы данных(hsqldb.org)
  hsqldb\bin\runManagerSwing.bat - gui клиент для работы с базой данных
  src                            - исходные коды программы
  target\PingPongServer.jar      - дистрибутив прогрыммы Ping'n'Pong server
  target\startServer.bat         - скрипт для запуска Ping'n'Pong и hsqldb серверов
  target\clientPage.html         - тестовая страница клиента

Команда для сборки проекта maven
  maven clean package

Написать сервер, реализовать 1 протокольную команду “PING <userId>”, на которую сервер должен отвечать “PONG N”, где N количество раз, сколько этот клиент запрашивал PING.
Сервер должен отвечать на POST запросы по адресу http://localhost/handler, тип команды и аргументы запаковывать в JSON внутри тела запроса. В архитектуру заложить то, что типов команд в будущем будет много (не только PING).
Хранилище данных выбрать соответственно задаче самостоятельно, учитывая, что сервис работает под высокой нагрузкой.
